What is the difference between relative, absolute and fixed positioning?

4 August 2024

I think the best way to explain positioning in CSS is by looking at it. Therefore, I’d rather keep this short and sweet by illustrating the differences between relative, absolute and fixed position through this webpage itself (which won’t be responsive to various screen sizes). I won’t be going into too much depth as it can get more complex than this.

To understand these concepts, it’s helpful to note that there is a ‘static’ position. A static position is a position where the element is in it’s natural html position on the page and will not move even if we use positioning elements to try and move it.

The Purple box is a our ‘Absolute Positioning’ example. When we don’t specify a static position, our elements are generally positioned absolutely by default. An absolute position is a essentially a position on the page where the element will go regardless of any other elements around, in or on top of it (assuming there is no z-index value assigned).
